git reset（走一个周期图）

供图

add , commit ,push都是常用的操作 不会出现太大的问题（等等说说push）

pull fetch在另一篇文章中有详细介绍。

mixed，hard，soft

  • HEAD

它保存着一个名为 HEAD 的特别指针。在 git 中，它是一个指向你正在工作中的本地分支的指针，可以将HEAD 想象为当前分支的别名。

  • index

Index空间，也叫做stage空间，或者cache空间

  • working tree

.git隐藏文件夹所在的文件夹，在本文中在工作区是gittest文件夹

我们输入git reset -h可以看到官网的解释

图操作流程

从这里我们能够看出 hard，mixed，soft的区别

  • hard

该参数用于在把工作副本改成一塌糊涂的时候，包括工作副本，一股脑恢复

  • mixed

把文件从缓存区撤销，恢复到指定的commit id但是会保存修改的内容，git reset默认的是mixed操作

  • soft

适合你commit之后突然想起来文件里面有不好的东西，撤回到add之后commit之前的状态。

当然，如果在缓存区域撤回到工作区间也是可以用reset的毕竟有mixed和hard。想象一下你不小心在文件中添加了不好的东西，这时候已经add进入缓存区了，这时候你也可以进行撤销操作
